REST

Breve Descripción de Rest

Ventajas de Rest
  1. Rest describe a cualquier interfaz que transmite datos específicos de un domino sobre HTTP sin una capa adicional, como hace SOAP. Existen una variedad de clientes que existen en la web, tales como, estaciones de trabajo, sistemas industriales, dispositivos móviles, etc. Lo que no degrada el rendimiento de rest.
Desventajas de Rest
  1. Rest No es un estándar, es tan solo un estilo de arquitectura, pero se basa en estandares tales como HTTP, URL, XML, etc.
¿Cuál es el formato de Rest?
  1. No se puede acceder directamente al recurso por lo que es importante obtener una representación que puede ser un documento HTML, XML o una imagen dependiendo de la situación. A continuación se ha usado XML como presentación, siendo el principal formato para intercambio de información.
